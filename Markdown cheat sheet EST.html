Markdown Cheat Sheet (eesti keeles)

Kiirjuhend Markdowni levinumate süntaksireeglite ja “GitHub-flavored” lisadega. Kõik näited on vormindatud code-stiilis, et saaksid need otse kopeerida.

Kiirviited

Pealkirjad Rõhutused Loendid Lingid Pildid Tsitaadid Kood Horisontaaljoon Tabelid Ülesandeloend Jalusemärkused Ankrud Erimärkide põgenemine HTML Markdownis Praktilised nipid

Märkused

Mõned funktsioonid (nt ülesandeloend, tabelid, jalusemärkused) sõltuvad platvormist. GitHub, GitLab ja paljud wiki-d toetavad “GFM” (GitHub Flavored Markdown) laiendusi.

Reavahetus: enamikus Markdowni variantides tähendab “uus rida” sama lõigu sees pehmet reavahetust. Kõva reavahetuse jaoks lisa rea lõppu kaks tühikut.

Pealkirjad Basics

# Pealkiri 1
## Pealkiri 2
### Pealkiri 3
#### Pealkiri 4
##### Pealkiri 5
###### Pealkiri 6
Nipp: jäta pealkirja ette alati vähemalt 1 tühik pärast #-märki.

Rõhutused Basics

Kaldkiri, paks, läbikriipsutus

*kaldkiri* või _kaldkiri_
**paks** või __paks__
~~läbikriipsutatud~~

Üla- ja alamindeks (platvormist sõltuv)

H<sub>2</sub>O (HTML-iga)
x<sup>2</sup> (HTML-iga)
Üla-/alamindeks ei ole standardne Markdown — kasuta vajadusel HTML-i.

Loendid Basics

Järjestamata loend

- Esimene
- Teine
  - Alam-element
  - Veel üks alam-element
- Kolmas

Järjestatud loend

1. Esimene
2. Teine
3. Kolmas

Järjestatud loend automaatse nummerdusega

1. Esimene
1. Teine
1. Kolmas
Alamloendi jaoks kasuta taanet (nt 2 tühikut või tab), kuid hoia stiil järjepidev.

Lingid Basics

In-line link

[Microsoft](https://www.microsoft.com)

Link koos pealkirjaga (tooltip)

[Microsoft](https://www.microsoft.com "Ametlik koduleht")

Viitelingid (reference-style)

Siin on [link][ref1] ja siin on teine [link][ref2].

[ref1]: https://example.com
[ref2]: https://example.org "Valikuline pealkiri"

URL “autolink” (paljudes variantides)

https://example.com

Põhisüntaks

![Logo](https://example.com/logo.png "Minu logo")

Tsitaadid Basics

Üherealine tsitaat

> See on tsitaat.

Mitmerealine tsitaat

> Esimene rida tsitaadist
> teine rida tsitaadist
>
> Uus lõik tsitaadi sees

Tsitaat koos loendiga

> Olulised punktid:
> - Punkt A
> - Punkt B

Kood Basics

Inline-kood

Kasuta käsku `npm install` sõltuvuste paigaldamiseks.

Koodiplokk (fenced code block)

```text
See on koodiplokk.
Siia võid panna käsud, konfiguratsiooni vms.
```

Süntaksi esiletõst (language tag)

```js
function tervita(nimi) {
  console.log(`Tere, ${nimi}!`);
}
```

Taandusega koodiplokk (vanem stiil)

    See on koodiplokk, mis algab 4 tühikuga.
    Sobib, kui fenced-plokke ei toetata.
Kui koodi sees on tagurpidi-ülakoma (`), võid inline-koodi ümber panna ka topelt- või kolmik-backtickid.

Tabelid GFM

Lihtne tabel

| Veerg A | Veerg B |
|--------:|:--------|
| Paremal joondus | Vasakul joondus |
| 123 | Tekst |
Joondus: :--- (vasak), ---: (parem), :---: (keskele). Väldi pikka koodi tabelis — pigem kasuta eraldi koodiplokki.

Ülesandeloend GFM

- [ ] Tegemata ülesanne
- [x] Tehtud ülesanne
  - [ ] Alamülesanne
Ülesandeloendid töötavad hästi GitHubi issue’de ja PR-ide kirjeldustes.

Erimärkide põgenemine Basics

Markdowni erimärkide “põgenemine” kaldkriipsuga

\*See ei lähe kaldkirja\*
\# See ei ole pealkiri
\> See ei ole tsitaat

HTML-entity’d (kindel viis)

&lt;div&gt;See näeb välja nagu HTML, aga ei renderdu.&lt;/div&gt;

HTML Markdownis Segarežiim

Ridade murdmine (HTML)

Esimene rida<br>
Teine rida

Keskjoondus / eripaigutus (HTML)

<div style="text-align:center">
  <strong>Keskel tekst</strong>
</div>
Mõned platvormid piiravad HTML-i turvakaalutlustel (nt skriptid, inline-stiilid).
